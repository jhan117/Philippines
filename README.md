## 2차 개발 (리팩토링 및 구조 개선)

### 2026년 7월 5일

**진행 내용:**

- 폴더 구조 개편 및 동적 라우팅 통합
  - 원인: 개별 페이지마다 폴더와 파일을 다 따로 만들다 보니 구조가 복잡해짐.
  - 해결: `(route)` 하위 폴더들을 다 삭제하고 `(subpages)/[...slug]/page.js`를 이용한 동적 라우팅 구조로 통합함. 컴포넌트 폴더도 `src/components`로 옮겨서 정리함.
- 수질 분석용 인포그래픽 이미지 직접 생성 및 교체 (wc1, wc3, wc4)
  - 원인: 기존 수질 상태 자료(오염 배출원 원형 차트, 지역별 합격/불합격 막대 차트 등)의 해상도가 지나치게 낮아 가독성이 매우 떨어졌음.
  - 해결: 파이썬 시각화 도구(matplotlib)와 DejaVu Sans 폰트를 활용해 논문 스타일의 고해상도 그래픽 데이터 이미지들(wc1.png, wc3.png, wc4.png)을 직접 생성하여 `public/images/waterQualityImgs` 하위에 업데이트함.
- 조원 사진을 아바타 이미지로 대체
  - 원인: 개인정보 노출을 방지하고, 기존 실물 사진 파일이 누락되면서 생긴 빌드 에러를 해결하고 싶었음.
  - 해결: 멤버 상세 카드 이미지 경로를 아바타 그래픽 파일로 일괄 변경함.
- Tailwind CSS 도입 및 CSS 모듈 제거
  - 원인: CSS 파일이 파편화되어 스타일 관리와 코드 중복 제거가 어려웠음.
  - 해결: 기존 CSS 모듈을 다 지우고 Tailwind CSS로 이관하여 마크업 코드를 심플하게 바꿈.
- 인터뷰 리스트 및 본문 페이지를 밝은 톤의 스타일로 변경
  - 원인: 기존 다크 테마 디자인이 사이트 전체 화이트톤 배경과 어울리지 않고 이질감이 듦.
  - 해결: 연한 회색 배경(`bg-slate-50`)과 주황색 포인트를 준 깔끔한 밝은 톤 매거진 스타일로 고침.
- 드롭다운 메뉴 Backdrop 닫힘 버그 수정
  - 원인: 메뉴를 연 상태에서 뒤로가기를 하면 드롭다운이 안 닫히고 불투명 배경(Backdrop)이 그대로 남음.
  - 해결: `usePathname`을 활용해 페이지 주소가 바뀔 때 드롭다운 상태를 false로 초기화하게 고침.
- 홈페이지 슬라이드 디자인 개선 및 블로그 이미지 조절
  - 원인: 슬라이드 전환 디자인 완성도를 높이고 본문 이미지 배치를 최적화하고 싶었음.
  - 해결: 홈페이지 슬라이드 디자인을 다듬고 본문 내 블로그 이미지 크기 세부 조정 작업을 완료함.
- 사이트 제목 및 헤더 로고 변경
  - 원인: 웹사이트 고유 브랜드 이미지를 정리하고 싶었음.
  - 해결: 타이틀 및 로고 텍스트를 'Philippines Insight'로 일괄 변경함.
- GitHub Pages 워크플로우 및 미사용 ESLint 설정 제거
  - 원인: 사용하지 않는 액션 스크립트를 정리해서 환경을 가볍게 만들고 싶었음.
  - 해결: nextjs.yml 배포 워크플로우 파일과 package.json의 homepage 설정, 안 쓰는 ESLint 플러그인 설정을 지움.

### 2026년 7월 8일

**진행 내용:**

- 홈페이지 메인 화면 전면 리뉴얼
  - 원인: 메인 화면 레이아웃을 고도화하고 다양한 카테고리를 보기 좋게 정돈하고 싶었음.
  - 해결: 사이트 소개 섹션인 `HomeIntro`와 카테고리 격자 목록인 `CategoryGrid` 컴포넌트를 새로 만들어 메인 레이아웃을 리뉴얼함. `HomeSlide`와 `HomeFooter`도 이에 맞게 수정함.
- 수질 정밀 지도 이미지(wc2) 및 문화 본문 음식 이미지 개선
  - 원인: 수질 페이지의 필리핀 행정 구역 지도(`wc2.png`) 해상도가 흐릿하여 알아보기 어려웠고, 문화 페이지 본문의 레촌 등의 음식 사진 화질이 떨어짐.
  - 해결: 수질 지도 이미지(`wc2.png`)를 히트맵 스타일의 고해상도 v2 원본 이미지로 되돌린 뒤 화면 레이아웃 비율(16:9)로 보정하여 새로 적용함. 문화 페이지는 고화질 검색 교체하여 로딩 에러와 함께 해결함.
- Next.js 이미지 qualities 경고 원인 분석
  - 원인: 콘솔에 unconfigured qualities(quality 100 경고)가 계속 뜸.
  - 해결: Next.js 기본 이미지 최적화 품질 배열(기본 75)에 우리가 쓰는 100 설정이 등록되지 않은 게 원인임을 규명함.
- Prettier 도입 및 전체 포맷팅
  - 원인: 파일 서식과 코드 스타일이 꼬여서 가독성이 떨어짐.
  - 해결: Prettier 패키지를 추가하고 `format` 스크립트를 만들어 전체 코드 스타일을 깔끔하게 자동 정렬함.
- 데이터 구조화 및 하드코딩 제거
  - 원인: 데이터 파편화로 중복 파일 관리가 생기고, 헤더나 카테고리가 수동 마크업으로 되어 있어 확장이 불편했음.
  - 해결: 중복 파일인 `contents.js`를 지우고 `pageConfigs.js`로 합침. `siteData.js`에 데이터를 모으고 헤더/푸터/카테고리 리스트에서 데이터 기반으로 동적 렌더링하도록 걷어냄. `MemberList.js` 내부 조원 ID 하드코딩 제거.
- 성능 및 로딩 성능 최적화
  - 원인: 폰트 로딩으로 인한 렌더링 지연과 많은 이미지/동영상 로딩으로 초기 부하가 큼.
  - 해결: layout.js에서 preconnect 링크 방식 대신 `next/font/google`을 사용해 Noto Sans KR 폰트를 불러오게 고침. 유튜브 링크를 데이터 설정(`videoUrl`)으로 분리해 lazy 로딩을 적용하고, `HomeSlide` 이미지 17개 로딩을 현재 및 좌우 인접 슬라이드만 동적으로 노출되도록 제어함.
- 사용자 경험(UX) 및 웹접근성(A11y) 강화
  - 원인: 스크롤 편의성이 낮고, 웹 표준에 안 맞는 마크업으로 스크린 리더 등 접근성이 저하됨.
  - 해결: 300px 스크롤 감지 시 헤더로 즉시 스크롤되는 플로팅 탑 버튼을 추가하고 상황에 따라 홈/뒤로가기 아이콘 분기를 처리함. `Back.js`, `Backdrop.js` 내 div 클릭 요소를 표준 `<button>` 태그로 변경하고 aria 속성 부여.

  - 원인: 모바일 화면에서 기존 상단 네비게이션이 가로로 늘어지며 오른쪽 영역에 불필요한 하얀 빈 화면(가로 스크롤 버그)이 발생하여, 모바일 디바이스에 최적화된 메뉴 레이아웃이 필요했음.
  - 해결: `HomeHeader.tsx`에 모바일 화면 대응용 햄버거 메뉴 버튼을 제작하고, 클릭 시 메뉴 리스트가 자연스럽게 내려오는 애니메이션을 적용함. 서브메뉴 클릭 시 하위 탭들이 아코디언 형태로 매끄럽게 펼쳐지며 목록 아이콘이 함께 표시되는 모바일 네비게이션 바를 구축함.

  - 원인: 모바일 화면 높이가 좁은 환경(또는 가로 모드)에서 햄버거 메뉴를 열었을 때 하단 메뉴들이 가려져 스크롤로 탐색할 수 없던 현상.
  - 해결: 모바일 네비게이션 래퍼에 `max-h-[85vh]` 높이 제한을 부여하고 내부 `nav` 요소에 `overflow-y-auto`를 설정하여 좁은 화면에서도 전체 메뉴를 자유롭게 스크롤하여 탐색할 수 있도록 사용성을 대폭 개선함.
- 데이터 설정 구조 분화 및 리스트 렌더링 수정
  - 원인: 문화, 날씨, 빈부격차 등 카테고리 본문 중 하이픈(`-`)으로 시작하는 목록 리스트가 줄바꿈 없이 한 줄의 일반 단락(`paragraph`) 텍스트로 처리되거나 데이터 구조가 꼬여 리스트 디자인이 깨지던 문제.
  - 해결: `pageConfigs.js`에 혼재되어 있던 데이터들을 `src/data/configs/` 하위 카테고리별 개별 파일들로 깔끔하게 정리하고, 목록 텍스트 구조를 정규 리스트 배열(`bullet-list`)로 마이그레이션하여 목록 들여쓰기와 bullet 기호가 정상적으로 출력되도록 수정함.
- 모바일 메뉴 닫기 시 Backdrop 잔상 버그 해결
  - 원인: 모바일 메뉴(햄버거 버튼)를 열고 내부 서브메뉴(Dropdown) 아코디언을 연 상태에서 메뉴를 닫으면, 데스크톱용 Backdrop 모달이 body 단에서 제거되지 않고 그대로 남아 터치가 차단되던 버그.
  - 해결: 모바일 햄버거 메뉴를 토글해 닫을 때(`isMobileMenuOpen`이 false가 될 때) 서브메뉴 활성화 상태(`openMenuId`)도 함께 `null`로 클리어해 주는 로직을 추가하여, 닫힘과 동시에 포털 Backdrop이 완벽히 언마운트되도록 수정함.
- 서브페이지 레이아웃 미니멀리즘화 및 가독성 개선
  - 원인: 회색빛 배경과 입체형 카드의 조화가 투박하고 가독성을 저해하였으며, 이미지의 테두리가 불명확하여 하얀색 이미지들이 배경과 경계 없이 묻히는 디자인 문제.
  - 해결: 본문 카드의 배경 및 패딩을 완전히 걷어내고 순수 화이트 배경(`bg-white`)과 자연스러운 여백을 적용하여 미디엄/브런치 스타일의 깔끔한 독서 환경을 연출함. 이미지 테두리에 `rounded-2xl border border-slate-200 shadow-sm`을 일괄 부여해 경계를 명확히 하고, 서브 네비게이션 버튼들에 보더 라인을 추가해 활성/비활성 명도 및 클릭 시인성을 높임.
